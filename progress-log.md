### Key Concepts Learned So Far
- **Git** is a save-point system (add, commit, push)
- **Staging area** = selecting which changes to include; **commit** = saving the snapshot
- **README** = front page of your project
- **.gitignore** = tells Git which files to skip
- **License** = tells others what they can do with your code (optional for learning)
- **Agent vs chatbot** = agents take actions, chatbots just answer
- **Skills** = reusable instruction playbooks for AI (SKILL.md files)
- **Terminal vs Agent** = use terminal for commands you want to learn, agent for bigger tasks
- **M / U / A** = Git file status indicators in Cursor (Modified, Untracked, Added)
- **Mermaid** = code that renders as diagrams in GitHub and Cursor

### Git Commands Practiced
- `cd`, `ls`, `git status` — navigation and checking state
- `git add`, `git commit -m`, `git push` — the full commit cycle (done multiple times)
- `git config --global` — setting up identity
- `git checkout -b`, `git checkout`, `git branch` — creating and switching branches
- `git merge` — merging branch changes into master
- `git diff` — seeing what changed in a file
- `git log --oneline` — viewing commit history
